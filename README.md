# Playwright Automation Framework

This repository contains an end-to-end automation framework built with Playwright and Allure reporting.

## Key Features

- Playwright test runner with browser projects for `chromium`, `firefox`, and `webkit`
- Allure reporting configured via `allure-playwright`
- Environment-specific base URL selection using `dotenv`
- Reusable test structure in `tests/`
- HTML reporter enabled by default

## Prerequisites

- Node.js installed (compatible with Playwright 1.58.2)
- `npm` available
- Allure CLI installed globally if you want to generate and open reports

Install Allure CLI:

```bash
npm install -g allure-commandline --save-dev
```

## Setup

1. Install dependencies:

```bash
npm install
```

2. Ensure environment variables are available in `config/.env`.

The configuration loads the `ENV` variable and maps it to one of the environment keys in `config/.env`:

- `QAENV`
- `STAG`

If `ENV` is not set, the default `QAENV` is used.

## Project Structure

- `playwright.config.js` - Playwright test configuration
- `tests/` - Test specification files
- `config/` - Environment configuration and `.env`
- `allure-results/` - Allure raw results output
- `reports/` - Generated Allure reports
- `playwright-report/` - Playwright HTML report output

## Available Scripts

### `npm test`

Run Playwright tests in headed Chromium:

```bash
npm test
```

### Smoke tests

Run core smoke tests and generate an Allure report:

```bash
npm run smoke
```

### Headed smoke tests

```bash
npm run smoke:headed
npm run smoke:headedChrome
npm run smoke:headedFirefox
npm run smoke:headedWebkit
```

### Regression tests

Run the broader regression test set and generate an Allure report:

```bash
npm run regression
```

### Staging environment

Run smoke or regression tests against `STAG` environment:

```bash
npm run smoke:stag
npm run regression:stag
```

## Test Execution Tips

- To run tests against a specific browser project, use `--project=chromium|firefox|webkit`.
- Playwright traces, screenshots, and video are configured for failures.
- HTML report is enabled by default and Allure output is saved to `allure-results/`.

## Customization

- Update `config/.env` to change environment base URLs.
- Add or modify tests inside `tests/`.
- Adjust browser/device settings in `playwright.config.js`.

## Notes

- The default base URL is loaded from `config/.env` using the `ENV` environment variable.
- `allure-results` is cleaned and regenerated by smoke/regression scripts before report creation.
